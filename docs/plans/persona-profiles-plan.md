# 人格プロファイル機能 — 詳細プラン & プリセット・マスター（マスター0）

> 目的: エージェントの「役割・部署・名前」に、世界的権威の**人格プロファイル**を1つ掛け合わせる仕様。
> 影響範囲: 口調・トーン・価値観・意思決定スタイル・口グセまで。**専門能力・成果物の正確性は役割/スキルが担保**（ペルソナはスタイルのみ）。
> 確定済み方針: マスタカタログ新設＋素(0番) / 1エージェント＝1ペルソナ / 実名OK。

---

## Part A. データモデル

### A-1. 新テーブル `persona_profiles`（マスタカタログ）

```sql
CREATE TABLE IF NOT EXISTS persona_profiles (
  id           TEXT PRIMARY KEY,          -- 'base' / 'warren-buffett' / uuid(カスタム)
  category     TEXT NOT NULL DEFAULT 'general',  -- finance/marketing/programming/legal/tax/accounting/management/design/founder/general
  is_base      INTEGER DEFAULT 0,         -- 1 = 素(0番)。プロンプト注入なし
  is_preset    INTEGER DEFAULT 0,         -- 1 = シード済プリセット（直接編集不可、複製のみ）
  enabled      INTEGER DEFAULT 1,
  sort_order   INTEGER DEFAULT 99,
  name         TEXT NOT NULL,
  name_ko TEXT DEFAULT '', name_ja TEXT DEFAULT '', name_zh TEXT DEFAULT '',
  one_liner_i18n TEXT,    -- 一覧用一行説明 {ko,en,ja,zh} JSON
  background_i18n TEXT,   -- 経歴/権威性 {ko,en,ja,zh} JSON
  traits_i18n  TEXT,      -- ★プロンプト注入する特性本文 {ko,en,ja,zh} JSON（口調/価値観/意思決定/口グセ/強み/注意）
  avatar_emoji TEXT DEFAULT '🎭',
  accent_color TEXT DEFAULT '#a855f7',
  tags_json    TEXT DEFAULT '[]',
  created_at   INTEGER DEFAULT (unixepoch()*1000),
  updated_at   INTEGER DEFAULT (unixepoch()*1000)
);
CREATE INDEX IF NOT EXISTS idx_persona_profiles_cat ON persona_profiles(category, enabled, sort_order);
```

> 命名は既存慣習（`name_ko/_ja/_zh`）に合わせ、長文（one_liner/background/traits）は i18n JSON で保持し `pickLang` で解決。

### A-2. `agents` への追加列（migration）

```sql
ALTER TABLE agents ADD COLUMN persona_profile_id TEXT REFERENCES persona_profiles(id);

```

### A-4. `traits_i18n` の構造（プロンプトに効く形）

各ペルソナの本文は以下の小見出しで構造化（Part B 参照）：
`口調 / 価値観 / 意思決定 / 口グセ・名言 / 強み / 注意`
※「注意」に必ず「専門能力・正確性は役割通り維持、スタイルのみ反映」を埋め込み。

### A-5. ★ペルソナを使わない設定（OFF制御・3層）

「プロファイル情報を一切使わない」状態を、**3つの独立した層**で明示的に制御できる。いずれか一つでも OFF なら、その対象ではペルソナはプロンプトに注入されない（＝従来挙動）。

| 層 | 仕組み | 用途 |
|---|---|---|
| **① 個別・素を選択**（既存） | `agents.persona_profile_id = 'base'` または `NULL` | そのエージェントにペルソナを付けない |
| **② 個別・適用トグル**（新規） | `agents.persona_enabled INTEGER DEFAULT 1` | ペルソナ割当を**保持したまま**一時的にOFF（後で戻せる） |
| **③ 全体マスタスイッチ**（新規） | `settings.personaInjectionEnabled`（既定 `true`） | 割当に関係なく**全社でペルソナ注入を停止**（機能ごと無効化） |

追加スキーマ:
```sql
ALTER TABLE agents ADD COLUMN persona_enabled INTEGER DEFAULT 1;  -- 1=適用 / 0=不使用
-- settings(KV): personaInjectionEnabled = "true" | "false"（未設定時は true 扱い）
```

判定ロジック（Part E のヘルパー内で一元化）:
```
全体スイッチ OFF        → 注入しない
個別トグル persona_enabled=0 → 注入しない
persona_profile_id が NULL/'base' → 注入しない（素）
上記いずれでもない        → ペルソナ traits を注入
```

> `personality`（自由記述）は従来どおり独立。空なら出ない。ペルソナOFFでも personality は使われる（両者は別レイヤー）。完全に「素」にしたい場合は ①〜③ のいずれか＋personality空。

## Part B. プリセット・マスター（マスター0 = 初期57レコード）

## Part E. プロンプト注入（中央集約）

現状8箇所の `Personality: ${agent.personality}` 直書きを、共有ヘルパーに集約（i18nの教訓）：

```ts
function buildPersonaPromptBlock(db, agent, lang): string {
  // ── A-5: OFF制御（3層）を一元判定 ──
  if (!isPersonaInjectionEnabled(db)) return "";        // ③ 全体スイッチ OFF
  if (Number(agent.persona_enabled ?? 1) === 0) return ""; // ② 個別トグル OFF
  const id = agent.persona_profile_id;
  if (!id || id === "base") return "";                  // ① 素 / 未割当
  const p = db.prepare("SELECT * FROM persona_profiles WHERE id=? AND enabled=1").get(id);
  if (!p) return "";                                    // 削除済/無効 → 安全に無注入
  const traits = pickLang(lang, JSON.parse(p.traits_i18n || "{}"));
  return traits ? `[Persona] ${localeName(lang, p)}\n${traits}` : "";
}

function buildAgentIdentityBlock(db, agent, lang): string {
  return [
    agent.personality ? `Personality: ${agent.personality}` : "",
    buildPersonaPromptBlock(db, agent, lang),
  ].filter(Boolean).join("\n");
}
```
注入順序: `[Task]/役割・スキル(能力)` → `Personality(個体)` → `[Persona](スタイル)＋ガードレール` → 言語指示(実装済)。
ペルソナOFF時はブロックが空文字となり、プロンプトは**従来と完全に同一**（コスト増ゼロ）。

## Part G. リスク・考慮点

- **実在名**: 「inspired by/〜の思想で」を明示し、本人の発言を断定的に騙らせない運用注記を traits「注意」に統一埋め込み（実装済の方針）。名言は一般に広く帰属される代表的なものを採用。実装前に1次ソースで最終確認推奨。
- **トークン肥大**: traits は簡潔に・上限文字数。素(base)は無注入でコスト増ゼロ。
- **8注入箇所のドリフト**: 中央ヘルパーに集約。
- **後方互換**: 既存エージェントは `persona_profile_id=NULL`＝素＝現行のまま。
- **preset保護**: 直接編集不可、複製してカスタム化。削除時は使用中エージェントを base へ戻す。
- **歴史的人物**: パチョーリ/ウィルソン等は「思想反映」と background に明示。
