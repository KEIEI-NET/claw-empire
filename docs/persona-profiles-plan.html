人格を掛け合わせる

「役割・部署・名前」に、著名な権威の人格プロファイルを1つ重ねる仕様。口調・価値観・意思決定スタイルまで反映し、専門能力・成果物の正確性は役割/スキルが担保。使う/使わないも完全制御できます。

◆ コンセプト

実効人格 = 役割/スキル（能力）× personality（個体の癖）× ペルソナ（スタイル）

現在:  エージェント = { 名前, 役割(senior), 部署(dev), CLIプロバイダ, personality(自由記述) }
新規:  上記 + 【ペルソナ・プロファイル】(1つ・任意)

例: 「イーロン・マスク型のシニア開発者」
   → コードは堅実に（役割/スキルが担保）、語り口と判断は大胆に（ペルソナ）
確定方針: マスタカタログ新設＋素(0番) / 1エージェント＝1ペルソナ / 実名OK / Persona Libraryはサイドバー新設 / 多言語はまず JA/EN。

A データモデル

マスタテーブル新設 ＋ agents への2列追加。

A-1新テーブル persona_profiles

CREATE TABLE IF NOT EXISTS persona_profiles (
  id           TEXT PRIMARY KEY,          -- 'base' / 'warren-buffett' / uuid(カスタム)
  category     TEXT NOT NULL DEFAULT 'general',
  is_base      INTEGER DEFAULT 0,         -- 1 = 素(0番)。注入なし
  is_preset    INTEGER DEFAULT 0,         -- 1 = シード済（直接編集不可・複製のみ）
  enabled      INTEGER DEFAULT 1,
  sort_order   INTEGER DEFAULT 99,
  name         TEXT NOT NULL,
  name_ko TEXT DEFAULT '', name_ja TEXT DEFAULT '', name_zh TEXT DEFAULT '',
  one_liner_i18n  TEXT,    -- {ko,en,ja,zh} JSON
  background_i18n TEXT,    -- {ko,en,ja,zh} JSON
  traits_i18n     TEXT,    -- ★プロンプト注入する特性本文 {ko,en,ja,zh} JSON
  avatar_emoji TEXT DEFAULT '🎭',  accent_color TEXT DEFAULT '#a855f7',
  tags_json    TEXT DEFAULT '[]',
  created_at INTEGER DEFAULT (unixepoch()*1000), updated_at INTEGER DEFAULT (unixepoch()*1000)
);
CREATE INDEX idx_persona_profiles_cat ON persona_profiles(category, enabled, sort_order);

A-5★ペルソナを使わない設定（OFF制御・3層）

「プロファイル情報を一切使わない」状態を、3つの独立した層で制御。いずれか1つでもOFFなら、その対象では注入されず従来と完全に同一のプロンプト（コスト増ゼロ）。

1
個別・素を選択（既存）— persona_profile_id = 'base' または NULL。そのエージェントに付けない。
2
個別・適用トグル（新規）— agents.persona_enabled。割当を保持したまま一時OFF（後で戻せる）。
3
全体マスタスイッチ（新規）— settings.personaInjectionEnabled（既定 true）。全社で機能ごと無効化。
全体スイッチ OFF          → 注入しない
個別トグル persona_enabled=0 → 注入しない
persona_profile_id が NULL/'base' → 注入しない（素）
上記いずれでもない          → ペルソナ traits を注入
personality（自由記述）は別レイヤー。ペルソナOFFでも personality は使われる。完全に素にしたい場合は ①〜③ のいずれか＋personality空。

function buildPersonaPromptBlock(db, agent, lang): string {
  if (!isPersonaInjectionEnabled(db)) return "";          // ③ 全体スイッチ OFF
  if (Number(agent.persona_enabled ?? 1) === 0) return ""; // ② 個別トグル OFF
  const id = agent.persona_profile_id;
  if (!id || id === "base") return "";                    // ① 素 / 未割当
  const p = db.prepare("SELECT * FROM persona_profiles WHERE id=? AND enabled=1").get(id);
  if (!p) return "";                                      // 削除済/無効 → 安全に無注入
  const traits = pickLang(lang, JSON.parse(p.traits_i18n || "{}"));
  return traits ? `[Persona] ${localeName(lang, p)}\n${traits}` : "";
}

function buildAgentIdentityBlock(db, agent, lang): string {
  return [
    agent.personality ? `Personality: ${agent.personality}` : "",
    buildPersonaPromptBlock(db, agent, lang),
  ].filter(Boolean).join("\n");
}

注入順序: [Task]/役割・スキル(能力) → Personality(個体) → [Persona](スタイル)＋ガードレール → 言語指示(実装済)。

ペルソナOFF時はブロックが空文字 → プロンプトは従来と完全に同一（コスト増ゼロ）。

  • トークン肥大: traits は簡潔に・上限文字数。素(base)は無注入でコスト増ゼロ。
